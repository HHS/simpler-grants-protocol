Core scalar types used throughout the protocol.

### isoTime

A time on a clock, without a timezone, in ISO 8601 format HH:mm:ss.

<Tabs>
  <TabItem label="JSON Schema">
  ```yaml
  $schema: https://json-schema.org/draft/2020-12/schema
  $id: isoTime.yaml
  type: string
  format: time
  description: A time on a clock, without a timezone, in ISO 8601 format HH:mm:ss
  ```
  </TabItem>

  <TabItem label="TypeSpec">
  ```typespec
  /** A time on a clock, without a timezone, in ISO 8601 format HH:mm:ss */
  @example(isoTime.fromISO("17:00:00"))
  scalar isoTime extends plainTime;
  ```
  </TabItem>

  <TabItem label="Example">
  ```json
  "17:00:00"
  ```
  </TabItem>
</Tabs>

### decimalString

A decimal number (with variable scale) encoded as a string, to avoid floating point issues.

<Tabs>
  <TabItem label="JSON Schema">
  ```yaml
  $schema: https://json-schema.org/draft/2020-12/schema
  $id: decimalString.yaml
  type: string
  pattern: "^-?[0-9]+\.?[0-9]*$"
  description: A decimal number (with variable scale) encoded as a string
  ```
  </TabItem>

  <TabItem label="TypeSpec">
  ```tsp
  /** A decimal number (with variable scale) encoded as a string, to avoid floating point issues */ 
  @pattern( "^-?[0-9]+\\.?[0-9]*$", "Must be a valid decimal number represented as a string" )
  @example("100.50")
  scalar decimalString extends string;
  ```
  </TabItem>

  <TabItem label="Examples">
  ```json
  // Scale 0
  "100"
  ```
  ```json
  // Scale 1
  "100.5"
  ```
  ```json
  // Negative, scale 2
  "-100.50"
  ```
  </TabItem>
</Tabs>
