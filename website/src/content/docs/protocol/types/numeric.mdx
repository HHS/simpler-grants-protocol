### numeric

Any numeric value, equivalent to `number` in JSON Schema. The base type for all numeric types.

<SchemaFormatTabs
  jsonSchema={{
    code: `
  $schema: https://json-schema.org/draft/2020-12/schema
  $id: numeric.yaml
  type: number
  description: Any numeric value
  `,
  }}
  example={{
    code: `
  // Scale 0
  100

// Scale 1
100.5

// Negative, scale 2
-100.5`,
}}
/>

### integer

A whole number without decimals, equivalent to `integer` in JSON Schema.

<SchemaFormatTabs
  jsonSchema={{
    code: `
  $schema: https://json-schema.org/draft/2020-12/schema
  $id: integer.yaml
  type: integer
  description: A whole number without decimals
  `,
  }}
  example={{
    code: `
  // Positive
  42

// Negative
-42`,
}}
/>

### decimalString

A decimal number (with variable scale) encoded as a string, to avoid floating point issues.

<SchemaFormatTabs
  jsonSchema={{
    file: { path: "website/public/schemas/yaml/decimalString.yaml" },
  }}
  typeSpec={{
    file: {
      path: "specs/lib/core/types.tsp",
      startLine: 42,
      endLine: 50,
    },
  }}
  example={{
    code: `
  // Scale 0
  "100"

// Scale 1
"100.5"

// Negative, scale 2
"-100.50"`,
}}
/>

### Other numeric types

TypeSpec supports several other numeric types, including `float` and `decimal`. A full list of these types can be found in the [TypeSpec documentation](https://typespec.io/docs/language-basics/built-in-types/#numeric-types).

:::caution
Because these other numeric types have limited support in JSON schema, they are NOT RECOMMENDED for use within a CommonGrants API specification.
:::
